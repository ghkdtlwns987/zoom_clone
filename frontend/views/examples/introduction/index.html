WebRTC 주요 API

  • GetUserMedia
    사용자의 카메라와 마이크 접근을 담당.
  • RTCPeerConnection
    Peer간의 연결을 생성하고, 오디오와 비디오 통신에 사용.
    (생성시 STUN 서버를 통해 Public IP 확인)
  • RTCDataChannel
    Peer간의 Data를 주고 받을 수 있는 Tunnel API
    (WebSocket과 유사하지만, P2P라 속도가 보다 빠름)

P2P기반 NAT 및 방화벽 통과 기법

  • STUN (Session Traversal Utilities for NAT) 서버
    P2P 통신을 위해 현재 클라이언트의 Public IP를 반환하는 역할을 하는 서버입니다.
  • TURN (Traversal Using Relays around NAT) 서버
    피어(Peer)간 직접(P2P) 연결이 실패할 경우 트래픽을 중계하는데 사용됩니다.
  • ICE (Interactive Connectivity Establishment)
    피어(Peer)간 통신을 위해 STUN, TURN 등의 기술을 통해 최적의 라우팅 경로를 찾아내는 기술입니다.
